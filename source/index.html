What's the problem with CSS?

CSS is naturally drawn towards chaos, disorder, and complexity.

{% markdown %} ```css .right { float: right; } ``` {% endmarkdown %}
{% markdown %} ```css .right, .rtl .left, .rtl #top .site-brand, .rtl #top .site nav, .rtl #top .site nav li, .rtl .cui-carousel-item, .rtl .case-study-group .block, .rtl fieldset label, .rtl .feature .feature-text, .rtl .feature .feature-block, .rtl .header .header-inner, .rtl .navigation-bar header, .rtl .navigation-bar nav, .rtl dl.inline dt, .rtl .toolbar .btn, .rtl .toolbar .dropdown-wrapper, .rtl .btn-group a, .rtl .input-list li label, .rtl input[type="checkbox"], .rtl input[type="radio"], .rtl .navigation-bar header h1, .rtl .navigation-bar-nav ul li, .rtl .tab-group .tab-nav .tab { float: right !important !important !important; } ``` {% endmarkdown %}

CSS has a tendency to get:

  1. Larger
  2. More Complex
  3. Harder to change
  4. Harder to maintain

It doesn't have to be terrible!

Lots of smart systems:

  • BEM
  • SMACSS
  • OOCSS

Preprocessors can help, too!

  • Stylus
  • Sass
  • LESS
  • PostCSS
  • Myth

But you don't need any of those to have better CSS...

1. Use low-specificity selectors

{% markdown %} ```css .sidebar {} /* less specific */ #page .sidebar div > a {} /* waaay more specific */ ``` {% endmarkdown %}

Measuring specificity

2. Don't use ID or element selectors

{% markdown %} ```css #banner { background-color: blue; } #banner h1 { color: white; } ``` {% endmarkdown %}
{% markdown %} ```css .banner { background-color: blue; } .banner-title { color: white; } ``` {% endmarkdown %}

3. Don't depend on a certain markup structure

{% markdown %} ```css .header ul li a { margin-right: 20px; } ``` {% endmarkdown %}
{% markdown %} ```css .header-link { margin-right: 20px; } ``` {% endmarkdown %}

4. Don't use inline styles

{% markdown %} ```html
Alert!
``` {% endmarkdown %}
{% markdown %} ```html

``` ```css .alert { background-color: red; color: white; border: 1px solid black; } ``` {% endmarkdown %}

5. Don't use !important

6. Prefix modifier classes

{% markdown %} ```css .btn { padding: .25em; background-color: blue; color: white; } .btn.red { background-color: red; } ``` {% endmarkdown %}
{% markdown %} ```css .btn { padding: .25em; background-color: blue; color: white; } .btn-red { background-color: red; } ``` {% endmarkdown %}

7. Write Small Rules

{% markdown %} ```css .widget { border: 1px solid #ddd; padding: 4em; background-color: #efefef; font-size: 4rem; margin-top: 24px; margin-bottom: 24px; } .gadget { border: 1px solid #ddd; padding: 4em; background-color: #efefef; font-size: 4rem; margin-top: 16px; margin-bottom: 24px; } ``` {% endmarkdown %}
{% markdown %} ```css .panel { border: 1px solid #ddd; padding: 4em; background-color: #efefef; } .font-large { font-size: 4rem; } .leader-1 { margin-top: 16px; } .leader-2 { margin-top: 24px; } .trailer-2 { margin-bottom: 24px; } ``` {% endmarkdown %}
